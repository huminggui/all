项目的开始

在webapck使用以下几个插件

postcss-loader
autoprefixer
babel-loader
babel-core
安装之后
创建两个文件
一个是.babelrc
一个是postcss.config.js

==以上两个文件的作用==
postcss的文件是用于处理css代码的，遇到需要加前缀的会自动加上前缀。主要用的插件是autoprefixer
postcss.config.js代码的实现如下：

const autoPrefixer=require("autoprefixer");
module.exports={
    plugins:[
        autoPrefixer()
    ]
}

.babelrc的文件主要处理vue中render函数以及遇到的jsx的代码。
关于.babelrc还需要安装一下几个插件：
babel-preset-env
bable-plugin-transform-vue-jsx

.babelrc的实现如下：

 {
   "presets": [
       "env"
   ],
   "plugins":[
       "transform-vue-jsx"
   ] 
}

然后
然后再对webpack中的.styl的规则进行更改，同时设置.jsx的规则，如下代码：

{test:/\.jsx$/,use:["babel-loader"]}, //这个是添加的

{
    test:/\.styl$/,
    use:[
        "style-loader",
        "css-loader",
        {
            loader:"postcss-loader",
            options:{
                sourceMap:true //作用是postcss-loader可以直接对stylus-loader的结果进行操作，会加快编译的速度。
            }
        },
        "stylus-loader"
    ]
},

可能会提醒我们还需要安装哪些插件，如下的插件是需要进行安装的，如下：
babel-helper-vue-jsx-merge-props
babel-plugin-syntax-jsx

todolist界面的实现

==v-for中key的好处==
提高效率，如果key相同则数据更新就不会再生成标签，而是根据key进行数据的更新。


颜色：大背景颜色,
#5E94A2 主背景色
red是主协助色
#304DBE 次协助色

vue中css添加背景图片

打包后的图片路径很明显是不对的。而且还不打包图片。
可以直接添加图片。
background:url(../) //这样的操作就很好。

css总html和body的margin和padding都设置为0了，为什么还有上边距

解决办法:
*{
margin:0;
padding:0;
}

其实是你里边的元素还有上边距，例如是否有h1等标签，如果有也设置为0，就没有了。

css中checkbox样式改，需要改默认样式，如下代码

    .todo li input[type=checkbox]{
       
       
       display: inline-block;
       text-align: center;
       vertical-align: middle; 
       line-
       position: relative
    }
     .todo li input[type="checkbox"]::before{
         content: "";
         position: absolute;
         top: 0;left: 0;
         background: #fff;
         width: 100%;
         height: 100%;
         border: 1px solid #d9d9d9
    }
    .todo li input[type="checkbox"]:checked::before{
        content: "\2713";
        background-color: #fff;
        position: absolute;
        top: 0;
        left: 0;
        width:100%;
        border: 1px solid #e50232;
        color:#e50232;
        font-size: 20px;
        font-weight: bold;
    }

webpack的配置优化

css打包出单独的文件

正常webpack打包的时候，css的文件是变成js的，这个时候的js文件是特别大的，浏览器缓存起来会比较困难。

extract-text-webpack-plugin 单独把css文件提取出来
另外是可以对css文件进行压缩的插件。postcss.config.js //这个还不是很会。
